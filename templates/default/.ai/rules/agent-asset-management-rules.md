# Agent Asset Management Rules

This project keeps human-maintained, cross-agent content in `.ai/` as the single source of truth.

## Canonical Source

Maintain shared assets in:

- `.ai/rules/`
- `.ai/skills/`
- `.ai/commands/`
- `.ai/workflows/`
- `.ai/entrypoints/`

Do not maintain the same shared content separately in tool-specific folders such as:

- `.agent/`
- `.agents/`
- `.claude/`
- `.codex/`

Those folders should be treated as generated targets, symlinks, or tool-specific containers.

## Official Entrypoints

Agent folder scanning is not equally reliable across tools. Official entrypoint files should be generated and kept current:

- `AGENTS.md` for Codex and other generic coding agents
- `CLAUDE.md` for Claude Code

Do not edit the generated shared-rules section directly. Update `.ai/entrypoints/project-context.md` or `.ai/rules/*.md`, then run the agent asset sync command.

## Shared vs Tool-Specific Content

Put stable, human-maintained content in `.ai/`.

Keep generated or tool-specific content in the tool directory that owns it. Examples:

- Tool-generated commands
- Local settings
- Cache folders
- Worktree state
- Skills generated by a third-party tool

## Updating Shared Assets

When changing shared rules, skills, commands, workflows, or entrypoint context:

1. Edit the canonical file under `.ai/`.
2. Run `pnpm sync:agent-assets` or `node scripts/sync-agent-assets.mjs`.
3. Check `AGENTS.md`, `CLAUDE.md`, and tool-specific target folders.

If `AGENTS.md` or `CLAUDE.md` has a tool-managed preamble before the generated title, the sync script must preserve it.
